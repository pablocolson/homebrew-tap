# pablocolson/homebrew-tap

Personal [Homebrew tap](https://docs.brew.sh/Taps) for command-line tools I
publish under [github.com/pablocolson](https://github.com/pablocolson).

## Formulae

| Formula | What it does | Source |
| --- | --- | --- |
| `tofu-summary` | Condensed, readable summaries of OpenTofu/Terraform plans. Reads `tofu show -json` and prints totals, per-type and per-module breakdowns, plus a heads-up on destroys/replaces of stateful resources. | [pablocolson/opentofu-summary](https://github.com/pablocolson/opentofu-summary) |

## Install

### One-line, no tap

```sh
brew install pablocolson/tap/tofu-summary
```

Homebrew auto-taps `pablocolson/homebrew-tap` and installs the formula.

### Explicit tap

If you'd rather add the tap once and install by short name afterwards:

```sh
brew tap pablocolson/tap
brew install tofu-summary
```

## Upgrade

```sh
brew update
brew upgrade tofu-summary                       # one formula
brew upgrade --formula pablocolson/tap          # everything in this tap
```

## Uninstall / untap

```sh
brew uninstall tofu-summary
brew untap pablocolson/tap                      # also drops every formula here
```

## Supported platforms

Formulae here ship bottles for the four combinations GoReleaser builds upstream:

- macOS — Apple Silicon (`arm64`) and Intel (`amd64`)
- Linux — `amd64` and `arm64`

Windows is not packaged. Use the raw binaries from each project's GitHub
releases or `go install` if you need them.

## How this tap is updated

`Formula/*.rb` is **auto-generated by GoReleaser** when a new version is
tagged on the upstream repo:

1. Upstream maintainer pushes a `vX.Y.Z` tag (e.g. on
   `pablocolson/opentofu-summary`).
2. The upstream release workflow builds platform binaries, creates a GitHub
   release with archives + checksums, then commits a refreshed formula here.
3. `brew update` picks up the new formula and `brew upgrade` installs it.

The commit history of this repo is therefore mostly bot commits authored by
`goreleaserbot`. Don't edit the formula files by hand — the next upstream
release will overwrite them.
